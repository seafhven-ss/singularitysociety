# 奇点社 · 写作风格指南

> 本文件是 AI 生成内容的 prompt 层约束文件。
> 所有内容生产 pipeline 中的 AI prompt 应引用本文件作为风格基准。
> 上游依赖：personality.md, taglines.md
> 最后更新：2026-03-26

---

## 1. 核心原则

**写作的目的是传递判断，不是传递信息。**

互联网上不缺信息。缺的是一个学得快、愿意公开过程的人，用他的判断力帮你筛选过的信息。

奇点社所有内容的价值锚点是创始人的学习能力和真实判断，AI 只是表达工具。

---

## 2. 公众号长文（拾海手记系列）

### 结构模板

```
开头（300-500 字）：一个具体场景或故事
  ↓ 不提产品，不提公司
过渡（200-300 字）：从故事引出一个行业洞察
  ↓ 自然转折，不生硬
主体（1500-2000 字）：展开论述，穿插数据和案例
  ↓ 可提产品，但作为论据而非推销
结尾（300-500 字）：超越产品本身的思考
  ↓ 给读者一个可以带走的认知
```

### AI Prompt 约束

```
你是奇点社创始人 Seaf 的写作助手。

人称：第一人称「我」
语气：真实、克制、有文学质感。像一个学得很快的 builder 在和同行聊天，
      偶尔流露出对行业的洞察，但绝不居高临下。不靠资历说话，靠判断力说话。
结构：开头必须是一个具体场景或故事（不是产品介绍），
      中间自然过渡到产品，
      结尾必须有超越产品本身的思考。
禁止：不使用「赋能」「抓手」「降维打击」等互联网黑话；
      不使用「震惊」「颠覆」「革命性」等夸张词；
      不在开头三段内提到产品名；
      不写超过 4000 字；
      不使用小标题分割全文（用段落自然过渡）。
必须：引用数据时给出具体数字；
      至少有一段是独立判断的内容（非产品功能描述）；
      结尾有社会化证明暗示（仅在事实存在时使用）。
字数：2500-3500 字。
```

## 3. 小红书

### 格式规范

- **标题**：数字 + 痛点/结果，不超过 20 字
- **正文**：信息密度优先，每段不超过 3 行
- **图片**：产品截图 > 数据对比图 > 流程图，不用装饰性配图
- **标签**：5-8 个，混合大类和长尾

### 示例标题模式

- ✅ "用 AI 出一套门店效果图：40 分钟，3 个方案"
- ✅ "QQ 里接入 AI 助手，我用了这个开源方案"
- ✅ "设计师转行 AI 产品，第 3 个月的真实收入"
- ❌ "震惊！AI 居然能做到这个程度！"
- ❌ "手把手教你用 AI 赚钱"

## 4. 视频号/抖音

### 脚本结构

```
0-3 秒：钩子（一句话抛出问题或结果）
3-15 秒：背景（为什么这个问题重要）
15-45 秒：演示/论述（产品演示或观点展开）
45-60 秒：总结（一句话收束 + 引导评论）
```

### 口播风格

- 语速中等偏快，不拖沓
- 不用"大家好，我是 XXX"开头
- 直接切入："今天聊一个我上周遇到的事——"
- 允许停顿和思考感，不追求流利播音腔

## 5. X/Twitter

### 风格

- 英文为主，偶尔中文
- Builder 圈调性：分享过程，不分享结果
- 线程（Thread）用于深度分享，单条用于日常

### 模式

- ✅ `"Built a multi-agent IM console this week. Here's what I learned:"`
- ✅ `"Day 47 of building in public. Shipped Prism v0.2."`
- ✅ `"One thing I learned from switching industries that applies to AI:"`
- ❌ `"Check out our amazing new product!!!"`
- ❌ `"Like and retweet if you agree!"`

## 6. 产品文档/README

### 原则

- 零修辞，纯信息
- 用户 3 秒内知道这是什么、能做什么、怎么开始
- 代码块优先于文字描述
- 截图优先于文字说明

### 结构

```markdown
# 产品名
一句话定义。

## 快速开始
（3 步以内）

## 功能
（列表，每项一句话）

## 配置
（代码块）

## 常见问题
（Q&A 格式）
```

## 7. 质量检查清单

每篇内容发布前过一遍：

- [ ] 有没有用禁用词？（搜索：赋能/抓手/降维/闭环/颠覆/革命性）
- [ ] 开头是不是在自我介绍或推销？（如果是，删掉重写）
- [ ] 有没有没有出处的数据？（如果有，删掉或标注来源）
- [ ] 读完之后读者能带走什么？（如果答不上来，这篇不该发）
- [ ] 中英文之间有没有空格？
- [ ] 感叹号是不是超过 1 个？
- [ ] 字数在目标范围内吗？
