# 官网改造质量审核报告

审核日期：2026-03-27  
审核范围：当前 `app/` 下新页面与相关站点壳层实现  
审核结论：**未通过，当前版本不建议进入 `dev -> main` 合并**

## 总结

本轮页面已经完成基础暗色化、产品页/案例页骨架和部分文案回填，但与 `data/website/design-spec.md` 的目标形态仍有明显差距。阻塞项集中在 4 类：

1. 路由与国际化架构未切到 `app/[locale]/`，`next-intl` 未接入。
2. 页面覆盖不完整，`/about`、`/contact` 缺失，导航也未补齐对应入口。
3. 产品页和案例页仍以占位面板替代真实截图/图片资产，图片优化链路实际未落地。
4. 多个 CTA 和信息点不符合设计规范或仍为占位目标。

## 1. 视觉一致性

判定：**部分通过**

已符合项：

- 暗色基底、边框、面板色和品牌渐变锚点基本落在 design token 范围内，见 `app/globals.css`。
- 首页、产品页、案例页都使用了较克制的卡片式工作台语义，没有出现霓虹网格、粒子、3D 几何体。
- 产品排序为 `TEBOT -> Nexus -> ProBrief -> Prism`，与阶段约束一致，数据定义在 `app/lib/site-data.ts`。

未通过项：

- 站点结构不完整。设计规范要求首页、产品总览、产品详情、案例总览、案例详情、关于、联系 7 类页面；当前仅有 `/`、`/products`、`/products/[slug]`、`/cases`、`/cases/[slug]`。
- 导航未达到规范。当前头部只有 `Products`、`Cases`、`Contact`，缺少 `About`、语言切换、移动端汉堡菜单，见 `app/components/site-shell.tsx`。
- Hero 仍是“产品与案例入口页”表达，不是规范里要求的品牌声明主导首屏。母品牌方法论被放进 `globalCards`，但没有升到首屏主叙事中心。
- 产品详情页使用的是伪工作台卡片，而不是规范要求的“真实产品截图 / 录屏容器”。
- 案例页缺少真实封面图、概览数据排布、结果数据模块，更像文字模板初稿。
- `TEBOT` 路由使用 `/products/tebot`，与设计规范里的 `/products/taoxiaofeng` 不一致。

## 2. 文案质量

判定：**基本通过，但未完全达标**

已符合项：

- 大多数中文文案符合“克制、短句、问题导向”的要求，没有出现禁用词如“赋能”“闭环”“革命性”。
- 产品定位与当前阶段信息基本和 Pipeline 输入一致。
- 文案主语总体克制，没有明显“我们如何如何”的宣传腔。

未通过项：

- 中英混排策略未系统落地。当前页面中只有少量英文眉标和视觉标签，不能视为完整英文站。
- ProBrief CTA 仍是占位链接 `https://github.com/`，不符合“每句话必须有信息量、每个入口必须真实可用”的要求。
- Footer 缺少 Telegram 二维码和完整联系组织，联系信息呈现不完整。
- 关于页品牌声明、创始人故事、IP 合作叙事、方法论模块尚未上线，因此无法完成该部分文案审核。

## 3. 响应式

判定：**基础可用，未达到规范要求**

已确认项：

- 当前页面主要采用移动端优先类名，常见区块在手机和桌面下能自然堆叠。
- 现有截图资产尺寸包含 `homepage-desktop.png` 1440x4200 与 `homepage-mobile.png` 430x4200，可覆盖桌面和手机取证。
- 产品页和案例页在 `lg` 以上切双栏，在移动端回落为单栏，基础布局逻辑正确。

未通过项：

- 没有针对平板断点的单独审核素材，本轮无法证明 `md` 断点表现达到要求。
- 移动端导航未实现汉堡菜单和全屏浮层，不符合规范。
- 规范要求“移动端不是桌面缩小版，是独立信息架构”；当前实现主要依赖同一套区块堆叠，缺少移动端专门裁剪。
- 首页产品入口在手机上仍是标准竖向列表，没有实现规范建议的横向滚动卡片或 2x2 独立组织。

## 4. 双语完整性

判定：**未通过**

阻塞项：

- 未采用 `app/[locale]/` 目录结构。
- 未安装或接入 `next-intl`，`package.json` 中也没有相关依赖。
- 当前仍残留旧的 `app/context/LanguageContext.tsx` 与 `app/constants/translations.ts`，说明国际化迁移未完成。
- 没有 `messages/zh.json`、`messages/en.json`。
- 页面文案主要直接写死在组件和 `app/lib/site-data.ts` 中，无法做 key 完整性校验。
- `html lang` 固定为 `zh-CN`，不存在英文版页面、语言切换、英文 URL 或消息回退机制。

结论：

当前站点不是“双语缺少少量 key”，而是**双语架构尚未建立**。

## 5. 性能与图片优化

判定：**未通过**

已确认项：

- 站点当前没有大图首屏背景、视频和重型前端动效，静态结构本身较轻。
- 代码中没有引入额外客户端状态库或复杂动画运行时。

未通过项：

- 当前页面没有使用 `next/image`。静态扫描未发现任何 `next/image` 或 `<Image />` 使用。
- 设计规范要求的产品截图、案例封面图、Telegram 二维码都没有真实接入页面，因此“图片已优化”这一项无法通过。
- `data/website/screenshots/` 下已有取证图片，但它们只是审核素材，不是站点运行时资源。
- `app/layout.tsx` 使用 Google Fonts 的 `next/font` 在线拉取字体。本地 `npm run build` 受网络限制直接失败，说明构建对外网字体依赖敏感，不适合作为稳态部署前提。

构建结果：

- `npm run lint`：通过，但有 2 条 warning，均来自已不应继续使用的 `app/components/IntroAnimation.tsx`。
- `npm run build`：失败。失败点是 `next/font/google` 拉取 `Inter` 资源时无法建立连接，导致 Turbopack 构建中断。

## 6. 链接检查

判定：**未通过**

已通过项：

- 现有内部链接 `/products`、`/cases`、`/cases/prism-launch`、`/products/prism` 与现有路由能对上。
- 产品卡片和案例卡片的 slug 生成逻辑正常。

问题项：

- 规范要求导航包含 `About`、`Contact` 内链，但当前路由和导航均缺失。
- Header 的 `Contact` 直接指向 `mailto:`，不是规范要求的 `/contact` 页面。
- Footer 没有 Telegram 二维码入口。
- ProBrief 的“查看 GitHub”指向 `https://github.com/` 占位地址，不是实际仓库。
- `TEBOT` 产品路由命名与规范定义不一致，后续若按规范补页面，会产生链接语义偏差。

## 建议的放行条件

达到以下条件前，不建议执行发布：

1. 完成 `next-intl` 迁移，建立 `app/[locale]/`、`messages/zh.json`、`messages/en.json`，清理旧 `LanguageContext`。
2. 补齐 `/about`、`/contact`，并把导航补到 `Products / Cases / About / Contact / LanguageSwitch / CTA`。
3. 将产品页、案例页、联系页的真实图片资源接入，并统一改用 `next/image`。
4. 替换所有占位链接，尤其是 ProBrief GitHub。
5. 明确 `TEBOT` 的最终 slug 是否保留 `tebot`，若不保留，统一为规范定义。
6. 解决构建对外网字体的依赖，至少保证 `npm run build` 在当前环境可重复通过。

## 审核结论

当前版本可视为“结构样机 + 文案初稿可用”，**不属于可部署版本**。建议继续留在 `dev` 分支整改，整改完成后再做第二次发布前审核。
